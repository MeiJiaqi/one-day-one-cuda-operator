# 🚀 Bonus Day: W4A16 Quantized GEMV (大模型量化解码核心)

## 📖 算子描述
针对 LLM 逐字生成阶段 (Decoding) 的极限带宽压榨技术。该阶段的计算主要为矩阵向量乘法 (GEMV)，性能瓶颈 100% 卡在 Global Memory 的权重读取上。
通过将权重从 FP16 (16-bit) 极度压缩至 INT4 (4-bit)，实现 75% 的物理显存节约。

**数学定义 (Dequantize-on-the-fly)**：
$$Y_{n} = \sum_{k=0}^{K-1} X_{k} \cdot (W_{n, k}^{int4} \times Scale_{n})$$

## 💻 核心优化纪实

* **V1 (朴素解包)**：使用 `uint8_t` 单字节访存与 `if` 分支解析。结果惨烈，解包指令开销远超节省的显存搬运时间。
* **V2 (向量化位运算)**：
  1. **宽位读取**：强行将内存指针 Casting 为 `uint32_t`，一次性吞吐 4 Bytes (8 个 INT4 权重)，访存指令数直降 75%。
  2. **无分支解析 (Branchless)**：利用位移 (`>>`) 与掩码 (`& 0x0F`)，配合三元运算符完成有符号映射，消灭了毁灭性的 GPU 分支预测失败。

## 🎯 终极战报 (16384, 8192)
在超大尺寸的真实 LLM 线性层对标中，V2 算子以 **0.1980 ms** 的成绩，成功反超 PyTorch 原生 FP16 的 **0.2760 ms**。
证明了在 Memory-Bound 场景下：**压缩数据的物理传输，永远是超越算力天花板的唯一解。**
